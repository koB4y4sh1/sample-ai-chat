# コミット規約

## 基本ルール

コミット案を作成する場合は、以下を日本語で書く。

- コミットタイトル
- 変更目的

## 推奨形式

```text
コミットタイトル:
<日本語の短いタイトル>

変更目的:
<この変更で何を達成したいのか>
```

## 書き方

- タイトルは変更内容が一目で分かる短い文にする。
- 変更目的は「なぜ必要か」を書く。
- 実装詳細だけを目的として書かない。
- 複数の目的が混ざる場合は、コミットを分ける。

## 例

```text
コミットタイトル:
モノレポの開発基盤を初期化する

変更目的:
FE、BFF、BE、MCPを同一リポジトリで管理し、依存関係と品質ゲートを統一するため。
```
