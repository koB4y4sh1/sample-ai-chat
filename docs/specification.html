Zenith AI Chat 開発環境仕様

作成日: 2026-04-24 / 対象: モノレポ初期開発環境

目的

FE、BFF、BE、MCPを1つのリポジトリで管理し、開発開始時点からformat、lint、 type check、pre-commitを同じ入口で実行できる状態にする。

構成

領域 ディレクトリ 技術 責務
FE a2ui TypeScript, React, CopilotKit, pnpm チャットUIとCopilotKit runtimeのフロント側統合。
BFF bff Python, FastAPI, uv FE向けAPI、認証・セッション・下流サービス呼び出しの集約。
BE ag-ui Python, AgentFramework, uv エージェント実行、会話状態、ツール利用方針の制御。
MCP mcp Python, FastMCP, uv エージェントから利用するツール群のMCP公開。

標準コマンド

目的 コマンド
FE依存関係の同期 pnpm install
Python依存関係の同期 uv sync --all-packages --dev
整形 pnpm run format
lint pnpm run lint
型検査 pnpm run typecheck
品質ゲート一括実行 pnpm run check
pre-commit有効化 uv run pre-commit install

開発プロセス

  • 日常開発はテスト駆動を標準とし、要求ごとに Red、Green、Refactor を小さく回す。
  • 層ごとのテスト責務、完了条件、契約変更時の手順は docs/tdd-specification.md を参照する。
  • scaffold 段階の package を除き、変更した挙動は import smoke test だけで完了扱いにしない。

品質ゲート

TypeScript

  • biome.jsonでformat、lint、import整理を統一する。
  • a2uiはtsc --noEmitで型検査する。
  • npm lockfileは使用せず、pnpm workspaceのルートロックに集約する。

Python

  • uv workspaceでbff、ag-ui、mcpを管理する。
  • ruffでlintとformatを実行する。
  • mypy strictで型検査する。
  • bandit設定はpyproject.tomlに集約する。

Git Hooks

  • pre-commitは基本検査、uv lock、Biome、Ruff、mypyを実行する。
  • hookはローカル環境のpnpmとuvを使う。

VS Code

  • .vscode/settings.jsonでBiome、Ruff、pytest、Python interpreterを定義する。
  • .vscode/tasks.jsonからinstall、dev、format、lint、typecheck、test、build、pre-commitを実行する。
  • .vscode/launch.jsonはFEのChromeデバッグを提供する。
  • BFF、BE、MCPは実装入口が未確定のため、launch設定はまだ定義しない。

GitHub

  • .github/workflows/ci.ymlでformat check、lint、type check、test、buildを実行する。
  • .github/dependabot.ymlでGitHub Actions、npm、uv依存を週次更新対象にする。
  • .github/pull_request_template.mdでPRごとの検証項目を固定する。
