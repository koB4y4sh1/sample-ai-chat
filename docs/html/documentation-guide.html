ドキュメントガイド

docs/spec の Markdown が正本です。ブラウザで読む場合は pnpm run docs:build 後に docs/html/*.html を開いてください。

まず読むもの（最短ルート）

  1. 全体仕様書 … アプリの目的・モノレポ構成・接続と Generative UI の結論。
  2. ワークスペース概要 … web / agent / mcp の境界とリクエストの流れ。
  3. 環境設定仕様 … ツールチェーン、セットアップ、環境変数、よく使うコマンド。

ここまでで「何のプロジェクトか」「どこに何があるか」「どう起動するか」が揃います。

ルートのドキュメントとの関係

  • AGENTS.md … コマンド必須セットとリポジトリ全体ルール。
  • CODING.md … 実装ルールの詳細（境界、CopilotKit TDD、テスト方針など）。

docs/spec は Zenith 固有の設計・運用の説明に寄せ、AGENTS.md / CODING.md と重なる部分は「要約 + リンク」に留めるファイルがあります（例: 開発ルール仕様）。

HTML の生成

pnpm run docs:build

出力先は docs/html/。仕様や索引を変えたあと、閲覧用 HTML をコミットする運用で揃えています。
